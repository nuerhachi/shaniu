# 膳你（ShanYu）

一个面向日常饮食记录与健康评估的平台。

## 1. 产品目标
在社交平台上，用户习惯分享早/中/晚餐图片与文字描述，但这些信息通常是零散、不可量化的。**膳你**希望把“分享”升级为“可追踪的健康数据”：

- 记录用户每日饮食（早/中/晚 + 加餐）
- 自动识别食物与营养素（热量、蛋白质、脂肪、碳水、钠、膳食纤维等）
- 按天/周/月评估饮食结构与健康风险
- 生成个性化饮食建议与可执行行动清单

## 2. 核心用户与场景
- **核心用户**：希望改善体重、血糖、血脂、肠胃健康的人群
- **典型场景**：
  - 每天拍照上传餐食并补充文字
  - 晚上查看当天营养分析分数
  - 每周接收“饮食报告 + 改进建议”

## 3. MVP 功能清单

### 3.1 饮食记录
- 快速记录：早/中/晚/加餐
- 支持图片 + 文本（如“鸡胸肉150g + 米饭半碗”）
- 可编辑份量（克数、份、碗、勺）

### 3.2 营养计算
- 解析食物并映射到食物营养库
- 自动汇总每日摄入：
  - 总热量
  - 三大营养素（蛋白质/脂肪/碳水）
  - 钠、糖、膳食纤维

### 3.3 健康评估
- 生成**每日饮食评分（0-100）**
- 维度：
  - 热量达标度
  - 宏量营养素平衡
  - 蔬果与纤维摄入
  - 高盐/高糖风险
  - 用餐规律性

### 3.4 个性化建议
- 当日建议：可执行到“下一餐”
- 周建议：长期结构性优化（如减少外卖、提高蛋白质占比）
- 风险提示：连续多天高钠/高糖时预警

## 4. 健康评分逻辑（可迭代）
建议初版采用可解释的规则引擎（便于上线与运营），后续再迭代为机器学习模型。

### 4.1 评分公式（示例）
`健康分 = 热量分(25) + 宏量平衡分(25) + 微量与纤维分(20) + 风险扣分(20) + 规律加分(10)`

### 4.2 规则示例
- 热量在目标区间（±10%）得满分，偏差越大扣分越多
- 蛋白质不足（< 推荐值80%）扣分
- 钠摄入超标（> 2000mg）显著扣分
- 连续 3 天缺乏蔬菜，触发“纤维不足”提示

## 5. 推荐系统思路
采用“规则引擎 + 模板生成 + 用户偏好过滤”三层策略：

1. **规则引擎**识别问题（如蛋白质不足）
2. **建议模板**生成可执行建议（如“明天午餐增加 1 份豆制品/鸡蛋”）
3. **偏好过滤**结合禁忌与口味（不吃辣、乳糖不耐受）

## 6. 数据模型（简化）

### User
- id
- 年龄、性别、身高、体重
- 目标（减脂/增肌/控糖/均衡饮食）
- 饮食偏好与禁忌

### MealRecord
- id, user_id, meal_type(早餐/午餐/晚餐/加餐)
- occurred_at
- image_url
- note

### MealItem
- id, meal_record_id
- food_name
- amount
- calories, protein, fat, carbs, sodium, fiber, sugar

### DailyAssessment
- id, user_id, date
- score
- risk_tags[]
- suggestions[]

## 7. 技术方案建议
- **前端**：Web（React/Next.js）或小程序
- **后端**：Python FastAPI / Node.js NestJS
- **数据库**：PostgreSQL
- **缓存**：Redis
- **对象存储**：S3/OSS（存餐食图片）
- **任务队列**：Celery / BullMQ（异步营养计算、周报生成）
- **AI 能力（可选）**：
  - 图片识别辅助食物识别
  - LLM 生成“更自然的建议文案”

## 8. 里程碑计划
- **第 1 周**：需求细化、原型图、数据表设计
- **第 2-3 周**：记录功能 + 营养计算 MVP
- **第 4 周**：健康评分 + 建议生成
- **第 5 周**：用户测试与指标验证（次日留存、周活、记录完成率）

## 9. 关键指标（North Star + 支撑指标）
- **North Star**：7 日内完成 ≥5 次饮食记录的用户占比
- 支撑指标：
  - 每日记录完成率
  - 建议点击率/采纳率
  - 连续记录天数
  - 饮食评分提升幅度（周环比）

## 10. 合规与隐私
- 健康数据属于敏感信息，需最小化采集
- 默认私密，用户可选择是否公开分享
- 账号删除时支持数据导出与彻底清除
- 明确声明：本产品为健康管理建议，不替代医疗诊断
